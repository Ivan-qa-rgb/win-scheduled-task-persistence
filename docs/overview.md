# Обзор кейса: Windows Scheduled Task Persistence

В этом кейсе показан один из базовых способов закрепления в Windows — создание запланированной задачи, которая запускает PowerShell‑скрипт с обходом Execution Policy. В качестве payload используется простой скрипт `svchost-update.ps1`, создающий файл‑артефакт `C:\Windows\Tasks\svchost-update.out`, что позволяет однозначно подтвердить успешное выполнение.

С точки зрения Red Team:

- скрипт кладётся в маскирующийся путь `C:\Windows\Tasks\svchost-update.ps1`;
- через `schtasks` создаётся задача `SystemUpdate`, запускающая `powershell.exe -NoProfile -ExecutionPolicy Bypass -File ...`;
- persistence проверяется запуском задачи и контролем наличия артефакта.

С точки зрения Blue Team / DFIR:

- задача обнаруживается и анализируется через `schtasks /Query /V`;
- выделяются подозрительные признаки (имя, команда запуска, обход Execution Policy, путь к скрипту);
- выполняется корректное удаление задачи и связанных файлов, что может лечь в основу плейбука для SOC и рекомендаций по детектам (мониторинг задач, запускающих PowerShell с Bypass и нестандартные скрипты).
